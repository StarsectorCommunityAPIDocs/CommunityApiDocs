013/**
014 * Extend this class instead of implementing ModPlugin for convenience if you do not
015 * intend to implement all the methods. This will also help avoid your mod breaking
016 * when new methods are added to ModPlugin, since default implemenations will be

018 *
019 * While having a ModPlugin for your mod is not strictly necessary, it serves as an entry point for all kinds
020 * of useful things, such as faction initialization, adding custom scripts and picking AI plugins.
021 *
022 * You need to instruct the game to load your implementation of this class by e.g. adding it to your mod_info.json:
023 * "modPlugin":"my.package.name.MyModBasePlugin"
024 *

048         * code in here gets executed before the game gets saved. This is mainly useful for doing cleanup.
049         * Anything that is part of the sector gets serialized when the game is saved. If you, for instance, add custom
050         * listeners to the sector and the game gets saved, they are added to the save file. If you
051         * then later e.g. rename the class of those listeners, that will break mod compatibility and users won't be able
052         * to upgrade your mod in an ongoing campaign. Also, players won't be able to remove your mod from an ongoing
053         * campaign. To avoid this issue, you can simply clean up anything that you added to the sector that doesn't need to
054         * be saved in this method and then re-add it in the afterGameSave-method.
055         *
056         * <p>Example:</p>
057         * <pre>
058         *     <code>
059         *                      while (Global.getSector().getListenerManager().hasListenerOfClass(MyListener.class)){
